# Muakkil — funnel launch kit (buildathon weekend)

> **What this is:** the [funnel/discoverability batch](../../../factory/playbooks/launch-stage/release-cadence-as-content.md) applied to Muakkil's launch. **Factory metadata only** — apply these to the muakkil.com landing / buildathon submission yourself; this file does not touch Muakkil's code (off-limits by hard rule). This is the *first product-execution* of the ingest loop (the funnel lessons get proven here).

## Honest fit note
Incumbent-anchoring ("the open-source X alternative") is a **weak fit** for Muakkil — it's a novel, aesthetic, ritual-UX product with no clean incumbent. So Muakkil's funnel should follow the **AutoGPT pattern, not the AppFlowy pattern**: lead with a *mesmerizing demo*, not a comparison. The Scribe flow is Muakkil's AutoGPT-grade asset.

## 1. The hero = the demo (do this first)
- **Lead the landing + submission with the Scribe demo:** voice → Scribe → Seeker → Herald → email, in **<60s**. A looping GIF/MP4 above the fold, autoplay muted. This is the single highest-leverage funnel asset Muakkil has.
- Capture it with `vhs` or a screen recording; keep it under ~20s for the loop (full run linked below it).

## 2. The 10-second line (under the demo)
Pick one — concrete over mystical for the cold visitor (keep the mystical voice for the body):
- **(Recommended)** *"Four AI agents that actually hand off to each other — speak once, watch the work move from agent to agent to your inbox."*
- *"A multi-agent workspace with a ritual UX — give one charge, four spirit agents collaborate to complete it."*
- *"Voice in, finished work out — Muakkil's four agents pass the task between them in under a minute."*

The mystical framing ("four spirit agents / charges / ritual UX") is your *differentiator and brand* — keep it, but **below** the line that makes a stranger understand the product.

## 3. Zero-friction trial
- One obvious CTA above the fold → live try at **muakkil.com** (no signup wall before the first "wow", if possible).
- First 100 sign-ups = beta cohort (already the plan). Make that explicit and scarce ("first 100 charter builders").

## 4. Release cadence (starts at launch, not after)
- Commit to a **visible weekly "what shipped" note** to the beta cohort (email + wherever the community gathers). Cadence is the content engine ([playbook](../../../factory/playbooks/launch-stage/release-cadence-as-content.md)).
- Each note = one content atom (cohort email + a post). Bank milestone posts (first 100, first cross-agent charge completed by a user, etc.).

## 5. Where to launch
- The buildathon itself + where this audience gathers (HN/Show HN, r/ClaudeAI, X). **Skip Product Hunt** — irrelevant for every winner in our study.
- An HN/Show HN hit is a *pulse* (~24h half-life), not the strategy — the demo + cadence convert the pulse.

## 6. Measure fit, not vibes (stars≠PMF)
Buildathon votes and sign-up spikes are **attention, not fit** (see [`stars-are-not-pmf`](../../../brain/anti-patterns/stars-are-not-pmf.md)). Track the beta cohort's **week-1 and week-2 retention** + a Sean-Ellis check once enough have used it. That's the real launch scoreboard.
