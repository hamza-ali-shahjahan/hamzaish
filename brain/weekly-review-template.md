# Weekly Review — Week of YYYY-MM-DD

Run every Monday morning. Takes 30 minutes. Output goes in `brain/decision-log/YYYY-MM-DD-weekly-review.md`.

## 1. Portfolio snapshot

For each of the 10 products, one line:

| Product | Stage | This week's #1 metric | Δ vs last week | Today's #1 action |
|---|---|---|---|---|
| ventbox | | | | |
| linkedup | | | | |
| ... | | | | |

(Generate this by running `/portfolio-pulse`.)

## 2. What moved

- Wins (3 max): what shipped, what worked, what surprised positively
- Losses (3 max): what didn't ship, what broke, what surprised negatively
- One non-obvious pattern across products

## 3. Where attention went last week vs where it should have gone

- Where it went: estimated hours per product
- Where it should have gone: what the portfolio-conductor would have said
- Gap analysis: 1 sentence on what to adjust this week

## 4. Decisions for this week

- 3 things to ship by Friday (one per product, max)
- 1 thing to deliberately NOT do
- 1 product to consider killing or pausing if no movement

## 5. Cross-product learnings

Anything that should go into `MEMORY.md`? If yes, write the file and update the index.

## 6. Next week's bet

One sentence: if I do only one thing next week, it's ___ because ___.
