# Learnings — 2026-07-07

## Ingested MetaHarness end-to-end under a goal contract — the rails made it slotting, not building

- Ran the full ingest of github.com/ruvnet/metaharness (a "factory for agent frameworks" — same species as Hamzaish) behind a written goal contract (`meta/goals/metaharness-ingestion.md`): 4 named metrics, 4 evals, anti-gaming guards, executed same session. Shipped as v2.9.0.
- **What worked:** writing the contract *first* made every later step mechanical — each artifact had a numeric bar before it existed (mine-items must path-resolve, knowledge must be brain-retrievable, the scanner needs a true-positive AND a false-positive fixture). The fresh-eyes verification prompt wrote itself because the contract already was one. This is `/write-a-goal` earning its keep on a non-build task; reuse the shape for future ingestions.
- **What surprised:** the factory needed *zero new rails* for this — `references/` peer-group section, the factory-change gate, the eval harness with its merge-safe `--update-baseline` (`{...baseline, ...passes}` — LLM-case PASSes survive a `--no-llm` update), and the learn-from-repos precedent (nanoGPT/D1) all existed. Ingestion has quietly become a *paved path*. The one real behavior change (MCP-config scanning in `/security-check`) took ~30 minutes because the eval harness already knew how to hold it to a floor.
- **The transferable rule (Darwin retention, now captured):** our factory-change gate covers *admission* (a change earns its place by a verdict); MetaHarness's Darwin Mode adds *retention* — a promoted rule whose named falsifier fires gets demoted automatically, not re-litigated. Every knowledge note written today carries an explicit "what would prove it wrong / revisit" line for exactly that reason.
- **Anti-pattern dodged, worth naming:** the temptation to add MetaHarness to `docs/LEARN-FROM-REPOS.md` — that syllabus is a counted, closed set of 200 guarded by CI; the mining home for live clones is `references/README.md`. Check which doc is the *canonical home* before appending to the one you found first.
