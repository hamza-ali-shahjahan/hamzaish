# 2026-07-01

## Researched the 2026 agent-framework landscape — found Hamzaish's real peer group

Triggered by a Gemini answer recommending CrewAI / Smolagents / Mastra as "AI
co-builders to learn from." Did a proper web sweep before answering. Two findings
worth keeping:

**1. The peer group I first named was wrong.** CrewAI / Mastra / LangGraph are
*runtime libraries* (the in-product agent loop). Hamzaish isn't that — its runtime is
the Claude Agent SDK. Hamzaish's actual peers are the **markdown-on-Claude-Code,
spec-driven systems**: BMAD-Method, Agent OS (Builder Methods), GitHub Spec-Kit,
Anthropic Skills. Those are a year-plus of public iteration ahead on some patterns and
are the right things to mine. (Cloned BMAD + Agent OS into `references/` same day.)

**2. Locked the terminology.** Hamzaish = an **agentic OS** built on the Claude Agent
SDK, *not* a runtime framework. Distilled into
`brain/anti-patterns/hamzaish-is-an-agent-os-not-a-runtime-framework.md` so a future
session doesn't (a) try to "modernize" the factory onto CrewAI/Mastra — a downgrade —
or (b) write "Hamzaish is a framework" into docs. README hero line stays as-is
(positioning-first); the precise category word already lives in `CLAUDE.md`.

**Surprise / signal:** the most useful thing wasn't the framework that "won" a ranking
— it was realizing Hamzaish already *is* the thing those frameworks sell, just
expressed in markdown + Claude Code. The leverage is studying the *prescriptive*
peers (BMAD/Agent OS), not adopting a *runtime* peer (Mastra) — and Mastra still earns
its place, but only *inside products*, never in the factory.

Runtime-default note for products that need in-app agent behavior: **Mastra**
(TS-native, ~19K stars, clean Next.js fit). Record as a `stack/` ADR when first used.
