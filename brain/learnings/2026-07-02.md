# Learnings — 2026-07-02

## Read a subsystem's own constitution before extending it — it may already forbid your plan

**Context:** Phase 1 of the trust-loop upgrade planned to backfill ~22 behavioral eval cases across flagship skills and Tier-A agents. Reading `meta/evals/README.md` *first* revealed two standing rules that killed that plan: every case must be verified honest-green against the live system before committing (mass authoring skips this), and the full LLM run already sits at its documented cost ceiling (4m55s vs. a 5-minute re-scope trigger — "new LLM cases must pay for themselves or replace one").

**Lesson:** the right design (a coverage *ratchet* with visible grandfathered debt + 2 verified cases) came from the harness's own documented constraints, not from the upgrade proposal. When a subsystem has a written constitution, the first implementation step is reading it — the audit-level view knows what's missing, only the subsystem knows what's *forbidden*.

**Where it's enforced:** `scripts/check-evals.ts` + `meta/evals/coverage.json` (the ratchet); rationale in `brain/decision-log/2026-07-02-eval-coverage-ratchet.md`.

## Audit inventories are leads, not facts — verify per-file before editing

**Context:** the full-factory audit (five parallel inventory passes) reported the 3 engineering agents as "no frontmatter — format outliers." A direct read before editing showed all three have correct frontmatter. One planned fix evaporated on contact with the file.

**Lesson:** a fan-out inventory is optimized for coverage, not per-file precision; treat its per-file claims as leads to verify, not edit instructions. Same family as `brain/anti-patterns/assuming-provenance-of-a-resolving-command.md` — check the artifact, not the report about the artifact.

**Where it's enforced:** habit + this entry; if it recurs, promote to an anti-pattern.

## Discipline gaps close with mechanical triggers, not stronger wording

**Context:** the audit found 1 retro and 1 decision-log entry for a year of heavy factory activity — despite both disciplines being clearly documented as mandatory. The fix that shipped: `scripts/check-retro.ts` (changelog entries ≥ 2026-07-02 must link a retro or record an explicit skip) and a decision-capture question wired into the end-of-session checklist and `/checkpoint`.

**Lesson:** when a documented discipline goes unfollowed for months, the document is not the problem and rewording it is not the fix — attach the discipline to an event that already always happens (a changelog entry, a checkpoint) and make silent skipping fail loudly. Skipping stays legal; skipping *silently* doesn't. This is the same shape as `check-validation`'s record-the-debt rule, now applied to retros and decisions.

**Where it's enforced:** `scripts/check-retro.ts` (CI) + `meta/factory-improving-factory.md` checklist + `factory/commands/checkpoint.md`.
