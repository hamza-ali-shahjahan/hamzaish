# Learnings — 2026-06-09

## Built the `web-launch` capability (verification-gated launch system)

Distilled a battle-tested launch workbook (14 phases, ~180 items, 24-item sign-off gate, Day 1/3/7/14/30 cadence, 18 gotchas) into a lean suite: `web-launch` spine + `launch-gotchas` + `pseo-at-scale`, plus a per-project workbook template. Core rule preserved: **Done ≠ Verified.**

### Surprise / reversal worth remembering: "plugin vs skill" hinges on *new-user setup*, not portability

I first recommended packaging as a plugin (portability: runs in any product repo). The operator pushed back through the lens of **"Hamzaish must be easy to set up for a new user."** That criterion flips the answer:

- **Skills** in `factory/skills/` are **zero-setup** — clone Hamzaish and they auto-discover via the existing `.claude/skills` symlink. A new user installs nothing.
- **Plugins** must be installed by **every** user, **every** machine (`/plugin` → add marketplace → install). Recurring friction, exactly what easy-setup fights.

I had under-weighted this because "forget the effort" was said earlier — but new-user setup ease is a *product property of Hamzaish*, not build-effort. Don't conflate them.

Also weakening the plugin argument: **`seo-aeo-bootstrap` already proves a skill can operate on a product's code from inside Hamzaish** (via `code-paths.local.json`). So "must be a plugin to run in the product repo" was a weaker claim than I made.

### The resolution that beat both: one source, two doors

Don't choose — and don't duplicate. Plugin folder = single source of truth; skills + command **symlink into it**. New users get zero-setup skills (default); the operator can `/plugin`-install the same folder for non-Hamzaish repos (opt-in). Edit once, both update. **Anti-pattern to avoid: two copies of the same skill = drift.** Codified the pattern in CLAUDE.md (`factory/plugins/`) and decision log.

### Dry-run on a real product earns its keep

Ran `web-launch` against `copyright`/Patently (live private beta → public launch). It worked *and* surfaced 3 concrete tooling gaps in one pass:
1. **The workbook leaned content-site, not app** — added a `site profile` (`app`/`content-site`/`hybrid`) toggle that pre-marks N/A blocks. Highest-value fix.
2. Stamp-location didn't fit Hamzaish — added "inside Hamzaish → `products/<slug>/launch/`."
3. Cosmetic `&lt;Project&gt;` → `[Project]`.

Lesson: **don't polish a tool before running it once on something real.** One dry-run > any amount of speculative refinement.

### Critical-recommendation muscle worked

Talked the operator *down* from 12 skills to ~4 by spotting that ~half my proposed "foundation crafts" duplicate existing plugins (`searchfit-seo:*`, `security-review`, `seo-aeo-bootstrap`). Default to checking what already exists before proposing new surface.
