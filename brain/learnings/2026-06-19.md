## The reusable insight

**A studied repo only improves you if its lesson is turned into a testable artifact with an ingest target.** "We should learn from X" changes nothing; "X teaches pattern P → here's the test that proves P fits us → here's the file P lands in" changes the factory. The discipline that makes the difference: every lesson gets (1) a concrete pass/fail test against our own repo, and (2) an exact destination file. Findings that fail are the highest-value output — they're the to-do list.
