# Learnings — 2026-07-05

## Story-first beats inventory-first in a README (competitor study: RedPlanetHQ/core)

**What worked (theirs):** a README sequenced promise → demo → thesis → concrete vignettes → one small component table, with define-by-negation ("Not a chatbot you open") and concrete-noun scenarios doing the selling. **What didn't (theirs):** the copy's claims run ahead of the public proof — a pattern to avoid, not just prose to admire. **Applied here:** README restructured story-first (hero negation line → demo → 11pm thesis → three vignettes → 6-row table, catalog collapsed into `<details>`), while keeping the inverse bet — every claim true and badged. Rule worth keeping: **counts are receipts, not pitches — proof presented before the claim reads as noise.**

## Present-tense aspiration hides in old docs — audit before repositioning copy

The go-live audit caught `stack/README.md` claiming "product #2 in ~5 minutes" when per-product wiring still takes ~25 min. The claim wasn't dishonest when written — it described account *signup* reuse — but drift between "accounts reused" and "provisioning repeated" turned it aspirational. Softened same day; `meta/goals/live-path.md` E3 earns it back with ledger receipts. Pattern: **whenever positioning copy is rewritten, audit the adjacent docs' quantitative claims against current reality — repositioning is when stale numbers get amplified.**

## "Local-first" copy can eat the actual ambition

The README sold local-in-60-seconds so well it read as the destination, while the repo's own chain (`/go-live` → `/security-check` → `/ship`) and the operator's vision aim at live, shareable, traffic-ready URLs. Fixed with one framing line ("Localhost is where products are born, not where they live") and by completing every local mention with its live continuation. Rule: **never let the free-and-instant on-ramp read as the product; always finish the arc in the same breath.**
