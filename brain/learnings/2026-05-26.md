## Hamzaish v1.0 restructure

**Setting**: Bootstrap session. Built `/Users/hamza/Claude/Hamzaish/` from the prior `Hamzaish the AI Cofounder/` scaffold + cloned references.

### What worked

- **Archive-first** (copy v0 verbatim into `_archive/v0/` before reorganizing) eliminated all "what if I broke something" anxiety. Recommended for all future restructures.
- **Symlinks survive `cp -R`** on macOS (default behavior preserves them). All 8 existing product symlinks transferred intact without re-creation.
- **Four-layer model (brain/factory/products/meta)** read cleanly to a fresh eye. The Founders Playbook 4 stages map naturally into `factory/agents/`, and the user's strategic concerns map to `brain/` and `meta/`.

### What surprised me

- **The three reference repos are designed to compose**: Garry Tan built gbrain as the brain for his openclaw + hermes deployment (`gbrain` ships an `openclaw.plugin.json`). I initially read them as 3 alternatives; they're 3 layers. This reshapes Phase E — when we add channels (openclaw) and a self-improving agent runtime (hermes), gbrain is the natural brain layer underneath them.
- **gbrain uses Bun + TypeScript**, matching the user's stack. That makes "wrap gbrain as a sibling process" (Phase C) much more viable than re-implementing.
- **The v0 `factory/` (skills + agents) was already substantial** — 8 skills, 5 stages of agents (idea/mvp/launch/scale/portfolio), a knowledge base with founders-playbook content, and a Next.js product starter. Roughly 70% of v1, not the 30% I'd guessed.

### What didn't work / friction

- **`factory/commands/` is empty.** v0 didn't have a top-level commands folder — slash commands lived as the `/hamzaish` alias in `~/.claude/commands/hamzaish.md` (global) and via skill auto-discovery. Decision: leave `factory/commands/` empty for now; project-scoped slash commands go here when needed. Not worth deleting.
- **The Founder's Playbook PDF** sat at the v0 root with no distilled markdown counterpart. Copied to `factory/playbooks/founders-playbook-source.pdf`; should distill into a markdown summary in Phase B (the agents reference `founders-playbook-distilled.md` but I didn't find that file in v0 — placeholder routing only).

## Phase A.5 — Foundational hardening

**Setting**: Same day. User pushed back on me wanting to dogfood Muakkil first. Compromise: build the foundational missing pieces (memory layer, entry-point wiring, broken-path fixes) before Muakkil, defer the rest of Phase B/C/D until after Muakkil's run produces real signal.

### What worked

- **Sub-agents for parallel audits** — Two Explore agents ran the 8-skill audit and 5-MVP-agent audit in parallel while I read the Founder's Playbook. Pattern worth keeping: when you need broad-but-shallow reads, parallel Explore agents trade ~30 seconds for the full picture.
- **Find-replace was risk-free at scale** — Once I verified the target paths existed (`factory/playbooks/mvp-stage/architecture-decisions.md`, etc.), a single `sed -i ''` across all `factory/agents/**` was the right move. ~30 seconds, zero broken links remaining.
- **FTS5 was a clear win over jumping straight to vectors** — 123 docs, 54ms full ingest, BM25 ranking that surfaces the right files. For personal-corpus scale (hundreds, not 146k), this is enough. Vectors stay as a Phase C investment only if FTS5 hits a ceiling.
- **Build-test-fix-test cycle on the memory layer** — Caught the hyphenated-token bug in the first smoke test (`work-on entry point` parsed as `work AND -on`, FTS5 read `on` as a column name). 5-line fix: wrap every token in double quotes. Wouldn't have found this without dogfooding the smoke test on real queries.

### What surprised me

- **The v0 work was deeper than my Phase A audit credited**. The Founder's Playbook IS already distilled. All 8 skills had clean YAML frontmatter and clear triggers. The 5 MVP agents had production-quality prompts. The "old folder is 30% of v1" estimate I started with was wrong — it was 70%+.
- **`bun:sqlite` is delightful**. Native, zero-install, the entire memory layer ingest + query is ~250 lines of TypeScript that runs natively under Bun.
- **Symlinking `.claude/commands/ -> factory/commands/`** is the obvious path for project-scoped slash commands when you also want them tracked as the canonical source. Works because git tracks symlinks as tiny files.

### What didn't / friction

- **Initial Edit tool error** — tried to Edit `mvp/builder/SKILL.md` before reading it. Cost 1 round-trip. Filed-under: always Read before Edit, even for surgical one-liner replacements. The tool refused; that's the discipline being correct.
- **The empty `factory/commands/` from Phase A was actually a real gap**, not just a placeholder. Without commands wired, slash commands wouldn't show up in `/`-autocomplete. Could have been caught earlier with one `ls .claude/commands/` smoke test.
