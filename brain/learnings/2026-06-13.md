# 2026-06-13 — Learnings

## Retiring feat/web-launch-plugin: port, don't merge

**Setting**: the long-lived `feat/web-launch-plugin` branch held ~820 lines of real work (the web-launch plugin) on a lineage that had diverged from main via an old rebase — same commits, different SHAs — while main had been rewritten 3× (README, CLAUDE.md, ledger) in the meantime.

### What worked
- **Direct-port instead of merge** for a diverged duplicate lineage: `git checkout <branch> -- <paths>` of only the additive/newer files onto a fresh branch off origin/main. Zero conflicts; a merge would have fought every file main rewrote. Verify "nothing unique left" with a tip-to-tip `git diff --stat` *before* deleting the branch.
- **Per-file verdicts, not wholesale trust**: 5 of the branch's modified files needed individual inspection — 3 were newer on the branch (momentum wording, CI hardening), 1 was newer on main (`factory-improving-factory.md` had gained the `/release` section), 1 was whitespace noise (MEMORY.md). Blind porting would have regressed main.
- **Counts-as-contract discipline held**: landing the plugin changed the README's "22 skills & commands" claim, so the same commit updated it to the recomputed 27. A repo that says "every count real" must treat any count-changing commit as a README-touching commit.
- **The staging worktree pattern** (separate worktree + rendered HTML preview + nothing committed until approval) survived its second full cycle, including being deleted and recreated mid-flight.

### What surprised
- **origin/main moved 3× during the session** (CODE_OF_CONDUCT, /release + v1.0.0, README "Builder Mode" edits) — multiple parallel sessions plus hand-edits on GitHub web. The fetch-rebase-push (never force) loop absorbed all of it, but every landing now requires a fresh `git fetch` + tip check immediately before push, not at session start.
- **Operator hand-edits on GitHub web are now a real input channel** ("Builder Mode" hero copy). Before touching README, diff the last few "Update README.md" commits to find and preserve them.

### Status: green — branch retired, plugin live on main (v1.19), repo down to one working branch + staging.

## Movement 1, brick #1: the harness shipped — and caught a real regression in its first 90 seconds

**Setting**: built the eval-harness de-risking slice (runner + check primitives + 3 deterministic `/brain-ask` retrieval cases), per `meta/evals/PLAN.md`'s gate.

### What worked
- **Bun.YAML exists** — case files stay YAML per the plan with zero dependencies.
- **Verify-the-expectation-first discipline**: all 3 queries probed against the live brain before authoring, so the baseline floor was honest-green on run one.
- **Self-testing the judge**: forced every verdict path before trusting it — FAIL_BUILDABLE (wrong expectation), GAP (no criteria), SKIP (missing env), and the regression floor (sabotage a baseline case → exit 1, restore → exit 0). A judge that has never caught anything proves nothing.
- **Blindness enforced in code, not convention**: `cases/`, `runs/`, and rubrics excluded from the brain index via path-prefix SKIP_PATHS in `brain/ingest.ts`. The first blindness audit caught a leak — a parallel session's `meta/evals/skills/go-live/rubric.md` was already indexed; name-based dir exclusion would have missed it (and collaterally hidden any product folder named `runs/`). Path-prefix exclusion is the right mechanism.

### What surprised (the good one)
- **The harness caught its own documentation as a regression.** Rewriting `meta/evals/README.md` (which documents the four-outcome verdict) changed the BM25 corpus — the README displaced PLAN.md to rank 4 for the verdict query, and the regression floor went red on the very next run. Retrieval was *correct*; the criterion was too narrow (the fact now lives in two canonical docs). Fix: widen `any_of`, criterion refined by a verdict. Lesson: **retrieval evals over a living corpus must expect fact migration** — criteria should name every legitimate source, and a red floor sometimes means "the corpus improved," which is still worth a human glance. That's `FAIL_BUILDABLE` doing exactly its job.

### Status: green — 3/3 PASS, floor established, judge blind, exit codes verified. Next gate: Hamza approves the shape → scale to 9 cases (/ideate, /validate need the LLM-judge seam filled).
