# Learnings — 2026-06-23

## Birthed `repolish` (README drafter + honesty pass) via the factory

**What worked**
- The factory flow held for a CLI product even though `/scaffold`'s default is a
  Next.js web app: used `/write-a-goal` to forge a measurable goal (M1 README
  completeness, M2 honesty recall/FP with robustness guards), then followed the
  *spirit* of `/scaffold` (metadata folder + decision log + validation ledger + goal)
  while hand-scaffolding a zero-dep TypeScript CLI as the code. Lesson: `/scaffold` is
  web-shaped; for CLI/library products, reuse its metadata convention but not its
  starter.
- Writing the eval **before** declaring done paid off — E1/E2/E3 turned "it works"
  into a number (acceptance MET: M1 100%, recall 100%, FP 0, 0 crashes on 5 real repos).

**Surprise (highest signal)**
- repolish's honesty pass flagged **its own README** — the line that *describes* the
  overclaims it catches ("production-ready", "tested on Windows") tripped the matcher.
  A claim-detector can't tell an assertion from a documentation mention of an example.
  Fix shipped: skip inline-code spans (`` `like this` ``) before matching, since
  backticked phrases are almost always examples/commands, not claims. General pattern:
  **any linter that reads prose will eventually lint its own docs about itself** — give
  it an escape hatch (code spans / fenced blocks) up front.

**Guardrail it became**
- Held the product fully LOCAL: code lives outside the always-public Hamzaish repo
  (registered in gitignored `code-paths.local.json`), its own git repo has no remote +
  a `.no-auto-push` marker. Reinforces the "nothing public until it's a deliberate
  step" rule even after a name is locked.

## Open question
- Pattern-based honesty recall is 100% on the seeded fixture but the real ceiling is
  unknown on wild READMEs — measure on ≥5 real ones before trusting the number (the
  LLM semantic pass is the lever that raises the ceiling; deliberately out of v0).

## Birthed `ship-guard` (local-first repo ransack-check + pre-push hook) via the factory

**What worked**
- Same factory flow as repolish, same day, and it held again: confirmed `/hamzaish`
  (= `/builder-mode`) is the front-door router; `/goal` is **not** a slash command —
  it's a *skill*, forged by `/write-a-goal`. Used `/write-a-goal` → `/scaffold`
  convention (metadata folder, not the Next.js starter) → `/build` for the zero-dep
  Node CLI. Reported the real commands rather than the assumed `/goal`.
- **Mining a real artifact for the spec beats guessing.** The check set
  (Anthropic `sk-ant-`, Clerk/Stripe `sk_live_`, Neon `npg_`/Postgres URLs, Supabase
  `service_role` JWT decode, Sentry `sntryu_`, private-key blocks, `pull_request_target`,
  unpinned actions) was reverse-engineered from `meta/security/REPO-ACCOUNT-SECURITY-AUDIT.md`
  — every pattern is something that actually bit the fleet, so recall hit 1.00 first try.
- Forged the goal with an anti-gaming guard (clean-fixture FP must be 0) so a
  flag-everything scanner can't fake high recall. Evals green: recall 1.00, 0 FP, install round-trip.

**Surprise — the self-scan trap recurred (now a confirmed pattern)**
- ship-guard hit the **same failure mode** repolish hit hours earlier: a tool that reads
  files for bad-strings will flag *its own source* (its secret regexes look like secrets).
  Two CLIs, same trap, same day → this is no longer a one-off. Fix: a `ship-guard:ignore-file`
  sentinel + default fixture/vendored-dir path ignores (repolish used code-span skipping).
  **Promotion candidate:** an anti-pattern — *"any scanner/linter that reads content will
  eventually match itself; give it a self-exclusion escape hatch from day one."* See the
  repolish entry above; if a third instance appears, write `brain/anti-patterns/`.

**Held LOCAL — and a layout mistake caught + fixed**
- First pass I nested ship-guard's code at `products/ship-guard/code/` *inside* the
  always-public Hamzaish repo. Wrong: the factory rule is **"a new product is its OWN
  repo; never paste product code into this repo"** (hamzaish.md guardrails). repolish got
  this right (own sibling repo `/Users/hamza/Claude/Repolish`, registered in gitignored
  `code-paths.local.json`); ship-guard didn't. Fixed by moving code to standalone
  `/Users/hamza/Claude/ShipGuard` (own git repo, **NO remote**, `.no-auto-push`), setting
  `code_path: null` + registering the path in `code-paths.local.json`. `products/ship-guard/`
  now holds **metadata only**. Held local until a deliberate publish step.
- **Lesson:** `/scaffold`'s `products/<slug>/code/` layout is fine for showcase metadata,
  but for a product meant to become its own public repo, scaffold the code as a *sibling*
  repo from the start — don't let `code/` live under the public Hamzaish tree. The split is:
  metadata→Hamzaish (committed, showcase), code→own repo (path in gitignored code-paths).
