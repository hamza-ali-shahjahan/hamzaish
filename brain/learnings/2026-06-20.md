# 2026-06-20 — Learnings

## Rot is a stage, not an accident — `/tidy` makes cleanup a first-class move

**Setting**: the `check-assets` guard caught a real bug (committed docs linking to gitignored `meta/research/` files — fine on the author's Mac, 404 for every reader and in CI). The operator asked: make the guard better, make rot-removal a *core stage*, and make it work across 100+ repos.

### What we learned
- **"Passes locally, fails for everyone else" is a whole bug class, not one bug.** A link resolves on the author's disk because the target is *gitignored* (present locally, absent in the repo); a relative path is off by one `../` (works in one editor's resolver, 404s on GitHub); a case-mismatched path passes on macOS, dies on Linux CI. The author's machine is the *least* representative environment. The fix is to check against **what's committed** (`git ls-files`, `git check-ignore`), not what's on disk.
- **Gate ≠ tool.** A *gate* (`check-changelog`, `check-assets`) must be narrow, fast, deterministic, and block every push. A *cleanup tool* (`/tidy`) is broad, slow (network dep checks), report-first, and run on purpose at a milestone. Conflating them makes the gate flaky and the cleanup toothless. Keep them separate: the gate guards the diff; the tool sweeps the repo.
- **Report-first is the safe default for anything that touches unfamiliar repos.** `/tidy --all ~/Claude` swept 26 repos and ranked them by rot without editing a byte. Across 100+ repos you've never audited, *seeing* the extent first — then fixing with per-change confirmation — is the only sane posture. A tool that auto-edits repos it's never seen is a footgun.
- **Summary-first beats detail-dump.** The operator's instinct: show the extent per category (🔗 / 🔑 / 🗑 / 📦) before any details, then recommend, then ask. A wall of 200 findings is ignored; a 4-line summary + "what next?" gets acted on.

### What shipped
- **`scripts/tidy.ts` + `bun run tidy`** — portable rot scanner (any repo; `--all <dir>` for many). Four categories: broken/gitignored links, committed-secret patterns (review-grade), orphaned assets, npm-unresolvable deps. Report-first; `--fix` is opt-in + confirmed.
- **`/tidy` skill** — the cleanup *stage* (`factory/skills/tidy/SKILL.md`): run → summary → recommend → ask → fix-with-confirmation.
- **`check-assets` hardened** — now flags references that resolve *only* to a gitignored target, so the original class is caught locally, not just in CI.

### Status: green. Tidy is portable + multi-repo; ran clean across the operator's machine. Next: wire `/tidy` into the momentum router's "milestone / pre-launch" moment; add case-sensitivity + `--fix` to the engine.
