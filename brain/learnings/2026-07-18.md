# Learnings — 2026-07-18

## The control plane is ports, not inventions

Every piece of v2.16.0 (orders directive, standing-orders authority table, hard spend cap,
states-and-dates gates) is a pattern someone else already paid scar tissue for — mined from
the `references/` clones and public harness-engineering practice, then re-implemented
Hamzaish-native. The study-then-port sequence made the build fast AND right-shaped on the
first pass. Rule reinforced: before building an orchestration mechanism, check
`references/` for who already learned its failure modes.

## Rendering absence beats exhorting presence

`check-gates`' first live render — 14 registered products, zero gates — did in one table
what months of "we should have kill criteria" intent did not: it turned the missing
discipline into the loudest fact in the repo, with a per-product fix path. Same mechanism
as the eval-coverage ratchet's debt list. When a discipline keeps not happening, build the
dashboard that shows its absence before building anything that enforces it.

## An unmeasured autonomous run is a decision, not a default

The autonomy loop now REFUSES unmeasured unattended runs (stream mode / cap 0) unless the
operator drops an explicit acknowledgement marker. The prior state — a warning nobody reads
scrolling past — is the pattern that produces $6k-overnight incidents elsewhere. Warnings
guard nothing; refusals with an explicit override guard everything and still respect the
operator's authority to choose.

## Parse tolerantly, meter as a floor

The spend meter depends on a CLI's JSON field (`total_cost_usd`) that has already been
renamed once (`cost_usd`). The parser accepts both, warns on failure, and records $0 —
explicitly documented as "the meter is a FLOOR, not the truth." External-format
dependencies inside safety mechanisms get tolerance + honesty about degradation, never
silent zeroes.
