# Learnings — 2026-07-08

## OpenWiki (LangChain, 2026-07-01) — the "thin router + diff-maintained wiki" pattern, adopted dormant

- Evaluated github.com/langchain-ai/openwiki (MIT): a CLI that generates an `openwiki/` docs folder for a repo, points `AGENTS.md`/`CLAUDE.md` at it, and — the part we lacked — ships a GitHub Action that re-reads the git diffs since its last run and opens a docs PR only for the pages that drifted. Shipped as v2.11.0: dormant workflow in the product starter, an "Agent docs from day one" section in `/scaffold`, opt-in steps in the starter's SETUP.md, practice #140 (⏳).
- **The design decision worth reusing:** ship automation *dormant*. The workflow gates every step on the `ANTHROPIC_API_KEY` secret existing, so a fresh scaffold stays true to the zero-config promise (runs green-and-silent) and opting in is one secret, not a config surface. Contrast with shipping it enabled: a daily red ✗ on every non-opted-in repo would train users to ignore CI. Same shape as the starter's env-gated integrations.
- **Secrets-rule interaction caught at design time:** `openwiki --init` writes a provider key to `~/.openwiki/.env` — that's a user-touched file per the 2026-07-03 rule, so `/scaffold` is explicitly forbidden from running the generation for the user. The skill says the sentence; the user runs the two commands.
- **Holding-pen status, honestly:** no Hamzaish product has run the loop yet. Unverified specifics to test on first real opt-in: does `openwiki --update` work when the wiki was generated on a different machine; is `claude-sonnet-5` an accepted `OPENWIKI_MODEL_ID` for the anthropic provider; does the opened PR's `add-paths: openwiki` miss instruction-file edits the tool makes outside that folder. Any of these failing → fix the workflow before evangelizing the practice.
- **What we deliberately did NOT do:** run OpenWiki on Hamzaish itself. The factory's docs are hand-crafted and are the product; a generated wiki would compete with them, and the tool auto-edits `AGENTS.md`/`CLAUDE.md` — our most load-bearing files. If ever tried: branch + review, never main.
