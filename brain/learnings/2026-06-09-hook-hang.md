### [SCORED] Global hooks ran unbounded git/network ops on every turn, in every repo

- **Cycle trigger:** Change to the factory itself (hook scripts hardened).
- **What happened:** Hamzaish's global Claude Code hooks (`scripts/auto-commit.sh` Stop hook, `scripts/auto-pull-rebase.sh` SessionStart hook) ran unbounded git/network operations (commit/push/pull) on **every turn in every repo** on the machine — no timeout, no fail-open, no scope gate. During a heavy build session this caused **repeated multi-minute session hangs**: each turn blocked on a git call with nothing to bound it, including in repos Hamzaish doesn't manage.
- **Scores:** Speed 5×2=10 · Build-quality 4×2=8 · Recurrence 5 · Generalizability 5 · Confidence 5 → **Composite 33/35**
  - *Speed 5:* the failure mode was a direct, repeated multi-minute stall on the critical path — removing it is decisive for build speed.
  - *Build-quality 4:* a wedged/killed turn risks half-applied state (e.g. a stale `index.lock`) and breaks build flow; the fix restores reliability.
  - *Recurrence 5:* fired on literally every turn and every session start, in every repo — maximally recurrent.
  - *Generalizability 5:* applies to every repo on the machine and to anyone who forks Hamzaish and wires the hooks.
  - *Confidence 5:* root cause is unambiguous (unbounded blocking ops in a global hook) and the fix is implemented + `bash -n`-verified.
- **Status:** PROMOTED → hardened hook scripts + anti-pattern guardrail
- **Promotion target:**
  - **Durable fix (code):** `scripts/auto-commit.sh` + `scripts/auto-pull-rebase.sh` — added a portable `run_with_timeout` shim (commit ≤10s, push/pull ≤20s), fail-open on timeout/error (stderr warning → `exit 0`), and an `is_hamzaish_managed` scope gate (Hamzaish repo itself / registered in `code-paths.local.json` / `.hamzaish-managed` marker). Opt-in push + secret scan preserved.
  - **Guardrail (so it can't recur):** `brain/anti-patterns/unbounded-git-in-global-hooks.md` — global hooks with blocking git/network ops require timeout + fail-open + scope gate.
  - **Docs:** `CLAUDE.md` §"Auto-commit + auto-push safety net" and `AGENTS.md` updated to describe all three properties.
- **Predicted gain:** Zero hook-induced session hangs going forward; the hooks act only on Hamzaish-managed repos; a slow/offline/hung remote can no longer block a turn (worst case: a one-line stderr warning, local commit kept).
- **Feedback check:** 2026-09 quarterly `/kill-or-keep` — confirm no hook-hang incidents recurred since the fix and that the scope gate didn't suppress wanted auto-commits in managed repos. Mark VALIDATED if clean; revisit limits/scope rule if not.
