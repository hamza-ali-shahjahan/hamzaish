# Shipping products with Claude Code — the practices ledger

**146 practices · 44 proven by real ships and incidents · 3 partially proven · 99 research-baked — every one sourced and linked.**

Most best-practice lists tell you how to *use* Claude Code. This one is about what comes after: taking a product from idea → MVP → launch → sell → scale with Claude Code as your cofounder — and not dying in the unglamorous parts.

**How a practice earns its badge here:** nothing on this page gets marked ✅ by opinion. Every session's lessons land in [`brain/learnings/`](brain/learnings/); at major-cycle boundaries, [`/learn-loop`](factory/commands/learn-loop.md) scores each candidate on five axes ([the rubric](meta/learning-loop-rubric.md), composite /35) and promotes only the top few into load-bearing guardrails; quarterly, `/kill-or-keep` re-checks every promoted practice — deliver the predicted gain or get sunset. The badges:

- ✅ **proven** — validated by a real ship or a real, dated incident in this factory (the product or wound is named)
- 🟡 **partially proven** — some of it validated here, the rest still waiting for contact with reality
- ⏳ **research-baked** — adopted from a named source or established practice; honest status: not yet battle-tested by one of *our* ships

The deep dives live in [`factory/playbooks/`](factory/playbooks/) (46 playbooks) — this page is the index that makes them scannable. For best practices on using Claude Code *itself*, [shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) is excellent — this ledger picks up where the code stops.

## ☠️ Never do this — each one cost us something real

- **Never create a new repo without checking the filesystem and existing remotes first.** 2026-05-30: 'publish hamzaish' was misread and a new PUBLIC agent-skills repo went live while Hamzaish already existed as a private repo. — ✅ *proven* · *Incident 2026-05-30 (agent-skills public repo; meta/retros/2026-05-30-wp-to-astro-shipping.md)* · [the incident](brain/anti-patterns/accidental-public-repo.md)
- **Never append inline # comments to a piped curl-to-bash one-liner.** Paxel, 2026-06-04: trailing comments mis-parsed on paste and SIGPIPE'd the script twice; identical comment-free lines ran clean. — ✅ *proven* · *Paxel incident, 2026-06-04* · [the incident](brain/anti-patterns/inline-comments-in-piped-bash.md)
- **Never wire unbounded git or network ops into a global Claude Code hook.** 2026-06-09: global auto-commit/auto-pull hooks blocked on git with no timeout or scope gate, hanging turns for minutes in every repo. — ✅ *proven* · *Incident 2026-06-09 (hook-hang; brain/learnings/2026-06-09-hook-hang.md, Composite 33/35, PROMOTED)* · [the incident](brain/anti-patterns/unbounded-git-in-global-hooks.md)
- **Never trust a dedup/upsert ingest's "rows inserted" log — count net-new rows.** 2026-06-16 (Patently Legal): CourtListener v4 search ignores `page=N` (it's cursor-paginated), so every page re-returned the same 20 results; `onConflictDoNothing` dropped them silently and the corpus sat frozen at 1,054 while logs scrolled "healthy" — misdiagnosed as a rate-limit for hours. Smoke-test that pages return distinct keys; pair idempotent writes with a net-new metric. — ✅ *proven* · *Incident 2026-06-16 (brain/learnings/2026-06-16.md)* · [the incident](brain/anti-patterns/silent-dedup-masks-broken-pagination.md)

- **Never put a product's code inside the factory repo (`products/<slug>/code/`).** 2026-06-23: ship-guard was scaffolded with its scanner inside the always-public Hamzaish repo; nothing gitignores `products/*/code`, so it was on track to be committed public (only `.no-auto-commit` saved it). Product code lives in its OWN sibling repo, registered in gitignored `code-paths.local.json`; `products/<slug>/` is metadata-only. — ✅ *proven* · *Incident 2026-06-23 (brain/learnings/2026-06-23.md; enforced by `bun run check-product-layout`)* · [the incident](brain/anti-patterns/product-code-inside-factory-repo.md)
- **Never put a pay-per-byte external scan (BigQuery/Athena) on a synchronous user path.** 2026-06-16 (Patently Legal): per-run `claim_similarity` scanned Google Patents BigQuery `claims_localized` (~370 GB/scan) per patent → **~$10.77 per clearance**. Pre-compute once into a local pgvector index, serve cosine locally → ~$0.10. A request that can trigger a metered scan is an unbounded bill. — ✅ *proven* · *Incident 2026-06-16 (brain/learnings/2026-06-16.md)*
- **Never cut a PR branch from a local `main` that's ahead of `origin/main` — branch from `origin/main` explicitly.** 2026-07-02 (PR #21): local main carried an unpushed parallel-session commit; branching off it smuggled that content into the squash-merge and published it unreviewed. The PR ships `origin/main..HEAD`, not your staged list — `git fetch origin && git switch -c change/<slug> origin/main`, then read `git diff origin/main --stat` before pushing. — ✅ *proven* · *Incident 2026-07-02 (PR #21 stowaway commit)* · [the incident](brain/anti-patterns/pr-branch-from-ahead-local-main.md)
- **Never commit with a placeholder git email — GitHub hands your commits to whoever owns that address.** 2026-07-03: a repo-local `user.email = noreply@users.noreply.github.com` override put a stranger's avatar (the real GitHub user "noreply") on our public repo — indistinguishable from an intrusion until reflog + push-activity proved it was us. Verify identity before pushing flows; the private address must carry YOUR `<id>+<username>` prefix. — ✅ *proven* · *Incident 2026-07-03 ("noreply from China"); enforced in setup step 2, auto-push refusal, /pr pre-flight* · [the incident](brain/anti-patterns/misattributed-commits-placeholder-email.md)
- **Never resolve an ambiguous instruction silently — declare your reading, or ask when stakes are real.** 2026-07-02: "go no rerun it" reads as *go on, rerun* or *go, don't rerun* — the session coin-flipped correctly on a state-changing command without flagging the fork; on a `git reset` instead of an idempotent script, that's an incident. Diverging readings + destructive/irreversible/external stakes → ask (readings listed, recommended first); otherwise take the momentum-preserving reading and say so. — ✅ *proven* · *Incident 2026-07-02 ("go no rerun it")* · [the incident](brain/anti-patterns/silently-resolving-ambiguous-instructions.md)
- **Never act on a fan-out report's per-file claim without reading the file.** 2026-07-02, twice in one audit: five parallel inventory agents reported "missing frontmatter" on 3 healthy files and a phantom TODO in a complete, tested runtime — both plausible, specific, and false; planned fixes evaporated on direct read. Aggregate findings degrade gracefully; per-file specifics are where confabulation concentrates. — ✅ *proven* · *Incidents 2026-07-02 (factory audit; /learn-loop 30/35 + fresh-eyes 27/35, PROMOTED)* · [the incident](brain/anti-patterns/trusting-fanout-reports-over-artifacts.md)
- **Never conclude a site is "fully static / no backend / scales trivially" from a keyless code read.** 2026-06-29 (Thousand Worlds Explorer): a scale audit of the keyless/dev state called it static with no DB — it actually had a live env-gated Supabase backend (auth, CRUD, telemetry, admin) plus a `/emulator` route proxying a separate app. The backend client only initializes when env vars are present, so a keyless read sees a local-only fallback. "Static" is a delivery claim, not a code-only property — verify the `.env` files, grep for backend SDKs (used conditionally), check hosting rewrites, then audit SMTP/DB-tier/RLS scale ceilings. — ✅ *proven* · *Incident 2026-06-29 (brain/learnings/2026-06-29.md)* · [the incident](brain/anti-patterns/concluding-no-backend-from-code-alone.md)

## 💡 Ideate & validate

- **Frame every problem as a job: When <situation>, I want <motivation>, so I can <outcome>.** People hire products to make progress in a circumstance; demographics and feature lists miss why they actually choose. — ⏳ *research-baked* · *Clayton Christensen (HBR articles) + Tony Ulwick's Outcome-Driven Innovation, refined further by Bob Moesta and Chris Spiek* · [playbook](factory/playbooks/idea-stage/jobs-to-be-done.md)
- **Before betting on customers switching, map push, pull, anxiety, and habit.** A switch only happens when push of the old plus pull of the new outweigh anxiety and inertia. — ⏳ *research-baked* · *The "Four Forces" framework: Moesta-Spiek interview methodology* · [playbook](factory/playbooks/idea-stage/jobs-to-be-done.md)
- **Serve the job's emotional and social dimensions, not just the functional task.** Most products only serve the functional dimension; functional-only positioning gets out-positioned by competitors. — ⏳ *research-baked* · *Clayton Christensen (HBR articles) + Tony Ulwick's Outcome-Driven Innovation, refined further by Bob Moesta and Chris Spiek* · [playbook](factory/playbooks/idea-stage/jobs-to-be-done.md)
- **Ask about specific past behavior, never hypothetical future intent.** People are bad at predicting their future behavior but excellent at describing what they actually did. — ⏳ *research-baked* · *The Mom Test by Rob Fitzpatrick (2013)* · [playbook](factory/playbooks/idea-stage/mom-test.md)
- **Talk for 15% or less of every discovery call.** If you're describing your product you're selling, not learning — the interviewee should do the work. — ⏳ *research-baked* · *The Mom Test by Rob Fitzpatrick (2013)* · [playbook](factory/playbooks/idea-stage/mom-test.md)
- **Score interviews by specific stories, named workarounds, and unprompted referrals — never compliments.** Compliments and "I'd totally use that" predict nothing; a referral is the most reliable validation signal. — ⏳ *research-baked* · *The Mom Test by Rob Fitzpatrick (2013)* · [playbook](factory/playbooks/idea-stage/mom-test.md)
- **Don't start customer discovery until who, frequency, severity, and workaround are all specific.** A vague hypothesis is unfalsifiable; a sharpened one can be killed cleanly in 5 interviews. — ⏳ *research-baked* · *Inspired by Y Combinator's "Idea Stage Validation" curriculum and Tony Ulwick's outcome statement format* · [playbook](factory/playbooks/idea-stage/problem-statement-rubric.md)
- **If there's no current workaround, doubt the problem exists.** Humans adapt; the workaround proves pain, is your real competition, and sets your price ceiling. — ⏳ *research-baked* · *Inspired by Y Combinator's "Idea Stage Validation" curriculum and Tony Ulwick's outcome statement format* · [playbook](factory/playbooks/idea-stage/problem-statement-rubric.md)
- **Triangulate market size with at least two methods — never top-down alone.** "1% of a giant number" is the classic anti-pattern; bottom-up forces a real customer count. — ⏳ *research-baked* · *factory playbook (follow-up sources it names: Andy Rachleff on market sizing; Crossing the Chasm — Geoffrey Moore)* · [playbook](factory/playbooks/idea-stage/tam-sam-som-templates.md)
- **Judge ideas by SOM, not TAM — an indie product needs roughly 10x its revenue target.** A $50B TAM means nothing if your obtainable slice is $50K or customers cost too much to reach. — ⏳ *research-baked* · *factory playbook (follow-up sources it names: Andy Rachleff on market sizing; Crossing the Chasm — Geoffrey Moore)* · [playbook](factory/playbooks/idea-stage/tam-sam-som-templates.md)
- **Wedge then expand: win a tiny niche 10x better before going horizontal.** Owning one niche completely beats being a day-one platform; expansion comes from the second user's variation. — ⏳ *research-baked* · *YC Startup School — Distilled Notes (Gary Tan era, 2024–2026)* · [playbook](factory/playbooks/idea-stage/yc-startup-school-notes.md)
- **For B2B, get 5 letters-of-intent or paid pilots before writing production code.** Selling first proves demand before the expensive build; validation beats engineering. — ⏳ *research-baked* · *YC Startup School — Distilled Notes (Gary Tan era, 2024–2026)* · [playbook](factory/playbooks/idea-stage/yc-startup-school-notes.md)
- **Win AI products on boring execution — auth, billing, support, reliability — not novelty.** Every AI product idea has 50 live versions; the winners out-execute the unglamorous parts. — ⏳ *research-baked* · *YC Startup School — Distilled Notes (Gary Tan era, 2024–2026)* · [playbook](factory/playbooks/idea-stage/yc-startup-school-notes.md)
- **Build the distribution motion alongside the product, not after.** Better product loses to better distribution every time; distribution is the moat. — ⏳ *research-baked* · *YC Startup School — Distilled Notes (Gary Tan era, 2024–2026)* · [playbook](factory/playbooks/idea-stage/yc-startup-school-notes.md)

## 🏗️ Build the MVP

- **supabase-js builders are lazy thenables — a `void insert(...)` sends NOTHING; end every fire-and-forget call in `.then()`.** ThousandWorlds shipped client analytics that never fired a single request (zero fetches to *.supabase.co, confirmed at the network layer); DB-trigger rows masked the dead logger. Verify telemetry with the network tab, never by reading code. — ✅ *proven* · *ThousandWorlds dead client analytics, 2026-07-12* · [the incident](brain/anti-patterns/fire-and-forget-supabase-builder.md)
- **Gunzip pre-compressed assets by magic bytes (0x1f 0x8b), never by file extension — and verify on both dev-preview and prod-like serving.** vite preview serves `.gz` with Content-Encoding: gzip (browser pre-decompresses → manual gunzip double-decompresses → silent ERR_ABORTED) while Vercel serves the same file raw. — ✅ *proven* · *ThousandWorlds tw-surface.u8.gz, 2026-07-12* · [the incident](brain/anti-patterns/trusting-gz-extension-over-magic-bytes.md)
- **Run every agent session stateful in writing, stateless in memory — docs are the memory.** Between sessions the agent remembers nothing; without written state Claude re-derives architecture every time and drift compounds until unmaintainable. — ⏳ *research-baked* · *Anthropic Founder's Playbook — MVP chapter on persistent context* · [playbook](factory/playbooks/mvp-stage/ai-native-dev-loop.md)
- **Give each session one topic; never mix refactors with feature changes.** Mixed intents destroy review-ability and the decision log captures neither change well. — ⏳ *research-baked* · *Anthropic Founder's Playbook — MVP chapter on persistent context* · [playbook](factory/playbooks/mvp-stage/ai-native-dev-loop.md)
- **Keep a decisions/surprises.md logging anything that behaved differently than CLAUDE.md implied.** Surprise entries accumulate into architecture corrections and expose where your written mental model has drifted from the code. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/ai-native-dev-loop.md)
- **Promote any architecture decision repeated across 3+ products into the factory starter template.** The next product gets it for free — this is the mechanism by which the factory improves itself. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/ai-native-dev-loop.md)
- **Write the ADR before the code — agentic coding compounds drift.** Without written decisions each Claude Code session re-derives the load-bearing choices and the codebase loses coherence. — ⏳ *research-baked* · *Michael Nygard's original ADR essay (2011): 'Documenting Architecture Decisions'* · [playbook](factory/playbooks/mvp-stage/architecture-decisions.md)
- **Every ADR needs a 'wrong if' section naming the signal that reopens it.** Decisions without a falsifier become dogma nobody revisits. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/architecture-decisions.md)
- **Supersede stale ADRs — never delete — and update CLAUDE.md to cite the replacement.** ADRs are the source of truth; CLAUDE.md is the working brief and goes stale silently if not re-synced. — ⏳ *research-baked* · *Michael Nygard's original ADR essay (2011): 'Documenting Architecture Decisions'* · [playbook](factory/playbooks/mvp-stage/architecture-decisions.md)
- **Lock the measurement framework before launch, never after.** Set afterwards, you cherry-pick metrics that flatter early traction. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/measurement-framework.md)
- **Pre-register three false-positive shapes that would look like PMF but aren't.** Launch spikes, founder-network signups, and novelty retention masquerade as fit unless distinguishing evidence is defined upfront. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/measurement-framework.md)
- **Benchmark retention on activated users, not signup cohorts, and watch for the flat tail.** A flat tail after D30 signals fit; a decaying tail is a leaky bucket — signup-cohort curves hide which one you have. — ⏳ *research-baked* · *Brian Balfour on retention curves: brianbalfour.com* · [playbook](factory/playbooks/mvp-stage/measurement-framework.md)
- **Write the three things the product deliberately does NOT do — the omissions are the product.** Successful products are 90% deliberate omission; the doesn't-do list is the positioning. — ⏳ *research-baked* · *April Dunford's Sales Pitch* · [playbook](factory/playbooks/mvp-stage/scope-document.md)
- **Amend scope only with quotes from 3+ real users who can't get value without the feature.** This evidence gate is the only loop that stops the feature creep that kills MVP products. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/scope-document.md)
- **Truncate the parking lot quarterly.** A 50-item parking lot is a dumping ground no one revisits — it stops being a decision tool. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/scope-document.md)
- **Send the PMF survey to active users only — recent login plus the activation action.** Surveying all signups dilutes the signal with people who tried, bounced, and have no opinion. — ⏳ *research-baked* · *Original Sean Ellis post: practicetrumpstheory.com (archived)* · [playbook](factory/playbooks/mvp-stage/sean-ellis-survey.md)
- **Segment the Sean Ellis score by persona and channel — a 50%+ segment is your wedge.** If one segment scores high while others lag, you've found who to double down on. — ⏳ *research-baked* · *Rahul Vohra's expansion (Superhuman PMF engine)* · [playbook](factory/playbooks/mvp-stage/sean-ellis-survey.md)
- **Mine the 'somewhat disappointed' 25-40% — they are the closest converts.** Their free-text answers name exactly what's missing to turn them into very-disappointed users. — ⏳ *research-baked* · *Rahul Vohra's expansion (Superhuman PMF engine)* · [playbook](factory/playbooks/mvp-stage/sean-ellis-survey.md)
- **Implement a per-user LLM spend cap before any user touches the product.** LLM bills spike fast; one runaway or abusive user becomes a cost incident overnight. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/security-checklist.md)
- **Verify Stripe webhook signatures on the raw body and dedupe by event_id.** Signature checks on parsed bodies silently fail, and replayed webhooks corrupt state without idempotency. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/security-checklist.md)
- **Derive subscription state from Stripe webhooks, never a local timer.** Local timers drift from billing reality; Stripe events are the source of truth. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/security-checklist.md)
- **Treat missing RLS, secrets in repo, or auth bypass as hard launch BLOCKs.** These are the only hard-stop verdicts; medium-and-below items can ship with dated backlog entries. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/security-checklist.md)
- **Validate env vars with zod at boot — fail fast on missing keys.** Missing keys surface at startup instead of mid-request in production. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/claude-md-templates/standard.md)
- **Tell the agent to refuse trivial tests — generate them only for auth, payments, and user data.** Test-everything bloats sessions; concentrating coverage where breakage hurts keeps the loop fast and the risk covered. — ⏳ *research-baked* · *factory playbook* · [playbook](factory/playbooks/mvp-stage/claude-md-templates/standard.md)

## 🤖 Build AI-native

- **Fence parallel agents by file ownership, keep them commit-less, and let ONE integrator own git.** Sixteen workflow subagents across 3 same-repo fan-outs held explicit fences perfectly; the integrator inventoried `git status` against reports 1:1 and committed with explicit paths — zero index races. Commit shared data assets BEFORE the fan-out. — ✅ *proven* · *ThousandWorlds phases 0–4 sprint, 2026-07-12* · [playbook](factory/playbooks/ai-native-2026/multi-agent-one-repo.md)
- **Write numeric gates into build-agent briefs: a self-consistency check, a determinism spec, and a real-UI cross-verification.** "Feed a real row through, expect ~N tolerance" caught a would-be unit bug and separated engine-intrinsic error (median 4.95 K) from mapping bugs (131 K); browser metrics later matched offline runs digit-for-digit. — ✅ *proven* · *ThousandWorlds emulator sync + validation panel, 2026-07-12* · [playbook](factory/playbooks/ai-native-2026/multi-agent-one-repo.md)
- **Before any architecture decision hinging on a performance guess, dispatch a measure-don't-estimate spike agent.** A 10-minute synthetic-model spike measured 0.78 ms worst-case GP inference in browser WASM (the folk 2–3× WASM penalty measured 1.0–1.5×) and turned a collaboration email from "should be feasible" into tables. — ✅ *proven* · *GP-LFR browser spike, 2026-07-12* · [playbook](factory/playbooks/idea-stage/landscape-research-before-roadmap.md)
- **Provision production auth before launch traffic — never ship on Clerk's dev instance.** Dev instances hard-cap at ~100 users, watermark the auth UI, and share Clerk's OAuth — the 101st launch signup fails. — ✅ *proven* · *Patently (IP Radar) go-live, 2026-06-09* · [playbook](factory/playbooks/ai-native-2026/auth-go-live.md)
- **Verify prod auth by the shipped pk_ prefix, never by local .env keys.** Local .env.local held dev keys while prod ran live keys; the stale-state misread cost a real debugging detour. — ✅ *proven* · *Patently (IP Radar) go-live, 2026-06-09* · [playbook](factory/playbooks/ai-native-2026/auth-go-live.md)
- **Treat auth webhooks as forward-only; they never backfill existing users.** Clerk webhooks fire only on new events — resolve pre-existing users on-demand via clerkClient in admin views. — ✅ *proven* · *Patently (IP Radar) go-live, 2026-06-09* · [playbook](factory/playbooks/ai-native-2026/auth-go-live.md)
- **Run name clearance as a hard gate before buying any domain.** patently.legal's domain was free, but the name collided with an 8-year-old patent platform — availability is not clearance. — ✅ *proven* · *patently.legal name-collision miss, cited in go-live design spec (2026-06-04)* · [playbook](factory/playbooks/ai-native-2026/go-live-provisioning.md)
- **Stop pipelines loudly on a stage failure — never silently continue.** The G06F21 silent-drop showed a silent continue hides data loss until far downstream; fail with a specific remediation instead. — ✅ *proven* · *G06F21 silent-drop (IP Radar build), cited in go-live design spec (2026-06-04)* · [playbook](factory/playbooks/ai-native-2026/go-live-provisioning.md)
- **Base64-encode multi-line secrets before pushing them to Vercel envs.** The Vercel CLI truncates multi-line stdin — learned the hard way during real provisioning. — ✅ *proven* · *Vercel CLI multi-line truncation, 'learned the hard way' — go-live design spec (2026-06-04), ties to deployment-learnings* · [playbook](factory/playbooks/ai-native-2026/go-live-provisioning.md)
- **Judge LLM output with a different model than the one that generated it.** Self-judging tilts positive — the generator excuses its own regressions; a separate judge model catches them. — ⏳ *research-baked* · *Hamel Husain eval essays; PromptFoo docs; Anthropic evaluation best practices* · [playbook](factory/playbooks/ai-native-2026/eval-driven-development.md)
- **Seed your golden dataset with cases where the AI previously failed.** Past failures are the highest-value evals — they encode exactly where your prompt is most likely to regress. — ⏳ *research-baked* · *Hamel Husain eval essays; PromptFoo docs; Anthropic evaluation best practices* · [playbook](factory/playbooks/ai-native-2026/eval-driven-development.md)
- **Give every eval a pass/fail threshold and run the suite on each prompt change.** Without a threshold an eval is just observation; without CI, prompts degrade silently as context and models change. — ⏳ *research-baked* · *Hamel Husain eval essays; PromptFoo docs; Anthropic evaluation best practices* · [playbook](factory/playbooks/ai-native-2026/eval-driven-development.md)
- **Treat a working prototype as a conversation prop, never as validation.** AI erased the cost of construction, not the cost of being wrong — a 3-day prototype can seduce you into 6 months of wrong building. — ⏳ *research-baked* · *Anthropic, The Founder's Playbook: Building an AI-Native Startup (2026)* · [playbook](factory/playbooks/ai-native-2026/founders-playbook-distilled.md)
- **Direct AI to disprove your idea, not validate it.** AI will validate whatever you ask it to; objectivity survives only by deliberately hunting disconfirming evidence. — ⏳ *research-baked* · *Anthropic, The Founder's Playbook: Building an AI-Native Startup (2026)* · [playbook](factory/playbooks/ai-native-2026/founders-playbook-distilled.md)
- **Write architecture into CLAUDE.md before any agent session, and append decisions after each.** Without specs, every Claude Code session re-derives foundational decisions — drift compounds into unmaintainable agentic technical debt. — ⏳ *research-baked* · *Anthropic, The Founder's Playbook: Building an AI-Native Startup (2026)* · [playbook](factory/playbooks/ai-native-2026/founders-playbook-distilled.md)
- **Call PMF only on a returning-paying-referring pattern across multiple iteration cycles.** Launch bursts from friends, investor portfolios, and HN spikes look like PMF and aren't. — ⏳ *research-baked* · *Anthropic, The Founder's Playbook: Building an AI-Native Startup (2026)* · [playbook](factory/playbooks/ai-native-2026/founders-playbook-distilled.md)
- **Declare a ship done only when live assertions pass, never when commands exit 0.** End every go-live with a scorecard of live checks (DNS, auth gate, build sha, secret leakage) — not a bare 'done'. — ⏳ *research-baked* · *Go-live provisioning design spec (2026-06-04), not yet built* · [playbook](factory/playbooks/ai-native-2026/go-live-provisioning.md)
- **Measure cost per accepted output, not cost per token.** If a cheaper model needs two retries to produce acceptable output, it cost more than Claude Sonnet on the first try. — ⏳ *research-baked* · *nousresearch.com / Hermes model cards; openrouter.ai; Anthropic prompt caching docs* · [playbook](factory/playbooks/ai-native-2026/hermes-and-fallback-models.md)
- **Check the prompt-caching math before switching to a cheaper fallback model.** Claude's caching savings can exceed Hermes' lower unit price — OpenRouter doesn't cache the same way. — ⏳ *research-baked* · *nousresearch.com / Hermes model cards; openrouter.ai; Anthropic prompt caching docs* · [playbook](factory/playbooks/ai-native-2026/hermes-and-fallback-models.md)
- **Wire MCP servers read-only by default; gate every mutation behind explicit user confirmation.** Write-scoped Stripe, Slack-post, or production-DB tools handed to an LLM are accidents waiting to happen. — ⏳ *research-baked* · *modelcontextprotocol.io spec; Anthropic Claude Code MCP docs* · [playbook](factory/playbooks/ai-native-2026/mcp-servers.md)
- **Cap MCP tool surface at 3-10 tools per server; 15+ degrades the model.** Too many tools confuse the model and every read adds latency — curate the stack per product. — ⏳ *research-baked* · *modelcontextprotocol.io spec; Anthropic Claude Code MCP docs* · [playbook](factory/playbooks/ai-native-2026/mcp-servers.md)

## 🚀 Launch

## 📈 Sell & scale

- **Never trust LIMIT to bound scan-billed query cost — set maximumBytesBilled.** BigQuery bills on bytes scanned, not rows returned; partition, cluster, and select only needed columns or one query blows the budget. — ✅ *proven* · *IP Radar BigQuery cost lesson, 2026-06-04 (brain/learnings/2026-06-04.md)* · [playbook](factory/playbooks/scale-stage/abuse-and-cost-controls.md)
- **Don't scale a channel until it converts repeatably with known numbers.** Ten customers from Twitter isn't channel proof; '100 cold emails/week, 5 convert, winning subject line known' is. Scaling before that burns founders out. — ⏳ *research-baked* · *factory playbook (follow-up: Reforge, Crossing the Chasm — Geoffrey Moore)* · [playbook](factory/playbooks/scale-stage/100-to-1000-customers.md)
- **By customer 300, automate, delegate, or eliminate every founder-only task.** Quality drops 10% but volume scales 5x — net positive. Founder-does-everything causes burnout at customer 200-400. — ⏳ *research-baked* · *factory playbook (follow-up: Reforge, Lenny Rachitsky)* · [playbook](factory/playbooks/scale-stage/100-to-1000-customers.md)
- **Track weekly cohort churn, not just MRR.** Annual contracts mask early churn until renewal year, and scaled acquisition quietly degrades new-cohort retention versus early cohorts. — ⏳ *research-baked* · *factory playbook (follow-up: Reforge, IndieHackers product interviews)* · [playbook](factory/playbooks/scale-stage/100-to-1000-customers.md)
- **Ship a no-redeploy kill switch for your most expensive feature.** When spend spikes at 3am, you flip one env var or feature flag and investigate in the morning instead of redeploying half-asleep. — ⏳ *research-baked* · *research-baked standard practice, awaiting first-incident validation (meta/RESEARCH-BAKED-PRACTICES.md)* · [playbook](factory/playbooks/scale-stage/abuse-and-cost-controls.md)
- **Enforce per-user AI caps server-side — the UI limit is only a suggestion.** A single abusive user or a looping bug can run a real bill in hours when every request costs tokens or metered API calls. — ⏳ *research-baked* · *research-baked standard practice, awaiting first-incident validation (meta/RESEARCH-BAKED-PRACTICES.md)* · [playbook](factory/playbooks/scale-stage/abuse-and-cost-controls.md)
- **Fix involuntary churn first: dunning recovers 50-70% of failed payments.** It's the cheapest churn to win back — card retries over 7-14 days plus Stripe's card updater, no persuasion required. — ⏳ *research-baked* · *factory playbook (follow-up: ProfitWell churn benchmarks, Reforge retention courses)* · [playbook](factory/playbooks/scale-stage/churn-reduction.md)
- **Spend on pre-cancel prevention, not win-back — prevention is 10x more efficient.** Behavioral triggers (14 days inactive, activity down 50% from baseline) catch churners while saving them is still cheap; win-back recovers only 5-10%. — ⏳ *research-baked* · *factory playbook (follow-up: ProfitWell churn benchmarks, Reforge retention courses)* · [playbook](factory/playbooks/scale-stage/churn-reduction.md)
- **If revenue churn exceeds logo churn, treat it as urgent — big customers are leaving.** Logo-only measurement hides which segment churns; dollar-weighted churn reveals whether you're losing small accounts (acceptable) or anchors (urgent). — ⏳ *research-baked* · *factory playbook (follow-up: ProfitWell churn benchmarks, Subscribed — Tien Tzuo)* · [playbook](factory/playbooks/scale-stage/churn-reduction.md)
- **Don't enterprise-prep before PMF; start SOC 2 only after first inbound enterprise interest.** Done in the right order it unlocks deals 10-100x SMB size; done out of order it eats months without revenue. — ⏳ *research-baked* · *factory playbook (follow-up: "Enterprise Ready" newsletter from WorkOS, Vanta/Drata SOC 2 prep content)* · [playbook](factory/playbooks/scale-stage/enterprise-readiness.md)
- **Sell with SOC 2 Type I plus an active Type II underway.** Most enterprise buyers accept this 'audit window'; waiting for Type II alone costs 12+ months and $20K-$50K before any deal. — ⏳ *research-baked* · *factory playbook (follow-up: Vanta/Drata SOC 2 prep content)* · [playbook](factory/playbooks/scale-stage/enterprise-readiness.md)
- **Never publish enterprise pricing; anchor at 3-10x your Pro tier, annual contracts only.** Negotiate up from the anchor on seats, SLA, and support tier; monthly enterprise contracts signal mistrust. — ⏳ *research-baked* · *factory playbook (follow-up: Founding Sales — Pete Kazanjy, Tomasz Tunguz blog)* · [playbook](factory/playbooks/scale-stage/enterprise-readiness.md)
- **Call it a loop only if the user's output reinvests as new-user input.** A funnel drawn with circular arrows doesn't compound; loops do — that's the entire difference, and the most common self-deception. — ⏳ *research-baked* · *Brian Balfour / Reforge (growth loops framework)* · [playbook](factory/playbooks/scale-stage/growth-loops-reforge.md)
- **Aim for a loop multiplier above 0.3, not virality above 1.** K > 1 is rare; most successful products run 0.2-0.4 loops that amplify other channels roughly 1.5-2x — that's winning. — ⏳ *research-baked* · *Brian Balfour / Reforge (growth loops framework)* · [playbook](factory/playbooks/scale-stage/growth-loops-reforge.md)
- **Fix the measured bottleneck step of the loop, not the one you intuit.** Quantify each step's conversion, cycle time, and friction first — the real constraint is rarely where you guess. — ⏳ *research-baked* · *Brian Balfour / Reforge (growth loops framework)* · [playbook](factory/playbooks/scale-stage/growth-loops-reforge.md)
- **Run the 30-day copy test: what survives a funded competitor's 1:1 clone?.** If your honest answer is 'better UX' or 'more features', you have no moat — AI commoditizes both within months. — ⏳ *research-baked* · *factory playbook (follow-up: 7 Powers — Hamilton Helmer, Anthropic's Founder's Playbook)* · [playbook](factory/playbooks/scale-stage/moat-building.md)
- **Pick 1-2 structural moats at Launch; deepen only those at Scale.** Ignore moats entirely at MVP (find PMF first), and never spread across all six — depth beats coverage. — ⏳ *research-baked* · *factory playbook (follow-up: 7 Powers — Hamilton Helmer, Anthropic's Founder's Playbook)* · [playbook](factory/playbooks/scale-stage/moat-building.md)
- **Measure lock-in in switching hours: under 4 is nothing, over 40 is a moat.** A concrete hours-to-switch estimate for a typical customer beats vibes when judging whether workflow lock-in is real. — ⏳ *research-baked* · *factory playbook (follow-up: 7 Powers — Hamilton Helmer)* · [playbook](factory/playbooks/scale-stage/moat-building.md)
- **Mitigate before you fix — a fast rollback beats a slow root-cause fix.** Stop the bleeding first (rollback, feature flag, maintenance page); fix forward only once users are unblocked. — ⏳ *research-baked* · *Google SRE Book, PagerDuty incident response, Supabase production checklist (research-baked, awaiting first-incident validation)* · [playbook](factory/playbooks/scale-stage/production-operations.md)
- **A backup you've never restored is a hope, not a backup.** Rehearse one restore to a fresh DB, confirm the app boots against it, write down the steps — that's your real DR plan. — ⏳ *research-baked* · *Google SRE Book, PagerDuty incident response, Supabase production checklist (research-baked, awaiting first-incident validation)* · [playbook](factory/playbooks/scale-stage/production-operations.md)
- **Use the pooled connection string (port 6543) for all serverless DB access.** Connection exhaustion from the direct 5432 string is the #1 self-inflicted DB outage on Vercel + Supabase. — ⏳ *research-baked* · *Google SRE Book, PagerDuty incident response, Supabase production checklist (research-baked, awaiting first-incident validation)* · [playbook](factory/playbooks/scale-stage/production-operations.md)
