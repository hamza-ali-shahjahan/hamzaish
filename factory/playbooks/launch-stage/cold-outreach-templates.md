# Cold Outreach Templates

Tested patterns. Use as starting points; personalize every send.

## Principles
1. Under 75 words. 50 for LinkedIn DM.
2. One ask. Not three.
3. Specific reference in line 1 (prove you read their work).
4. No links above the ask.
5. From a real human (no `noreply@`, no signature graphics).

## Discovery interview ask (cold email)

```
Subject: <pain> question

Hi <name>,

<one sentence proving relevance — quote their LinkedIn post / blog / repo>.

I'm researching how <role> handle <specific problem>. Not selling anything — just learning.

Got 15 min this week? Happy to share what I find.

— <name>
```

## Early customer ask (post-prototype, pre-launch)

```
Subject: <outcome verb> for <segment>

Hi <name>,

<one-line relevance>.

I built <product>: <one-liner>. A few <segment> are using it now, getting <specific outcome>.

Worth 10 min? Would love your feedback even if not a fit.

— <name>
```

## LinkedIn DM — discovery

```
Hi <name> — saw your post on <topic>. Curious how you currently handle <related pain>? Doing user research, 10 min if open.
```

## LinkedIn DM — sale

```
Hi <name> — saw <relevance>. Built something for <pain>. Different from <obvious competitor> in that <one-line diff>. Worth a look? Happy to demo or send a video.
```

## Follow-up #1 (T+4 days)

```
Hi <name> — bumping this in case it got lost. Even a "no thanks" helps me prioritize. Thanks either way.
```

## Follow-up #2 (T+10, breakup)

```
Hi <name> — last from me. If this isn't relevant, no worries. Easy to find if it ever is.
```

## Subject line patterns (tested)

| Pattern | Example | Best for |
|---|---|---|
| `<pain> question` | "Contract review question" | discovery |
| `<topic> for <segment>` | "AI form-building for SaaS founders" | sale |
| `quick <Y>-min ask` | "quick 10-min ask" | warm intro |
| `<their name in subject>` | "Sarah - quick one" | very high reply rate, sparingly |
| `<mutual interest>` | "Loved your post on RLS" | when there's genuine context |

Avoid:
- `Following up` (generic)
- `Re: <made-up thread>` (manipulative + spam-flagged)
- `Quick question` (overused)
- `Have you seen ___?` (clickbait)

## Opening lines that work

- "<Specific thing they did> — really sharp take on <X>."
- "Reading your blog post about <X> — had a question about <Y>."
- "Saw you're hiring for <role> — guessed you're dealing with <related pain>."
- "<Mutual contact> mentioned you've been wrestling with <pain>."

## Opening lines that don't

- "Hope this finds you well"
- "I hope you're having a great Monday"
- "I'm reaching out because..."
- "I came across your profile..."

## Cadence rules

- Send 25 messages/day max — quality > volume
- Test ONE variable at a time (subject, opener, ask)
- 2 follow-ups max — stop after that
- After 100 sends: if reply rate < 5%, fix the message or the audience (don't grind through)
- Track in a sheet: name, channel, date, subject, reply (yes/no/no-reply/negative), outcome

## A/B test framework

After 50 sends per variant, you can compare. Variables worth testing:
- Subject line (biggest delta)
- Opener (one specific reference vs generic)
- Ask (15 min vs "send me a video reply")
- CTA wording ("got time?" vs "would you be open to?")
- Send day (Tue/Wed best; Mon/Fri worst)
